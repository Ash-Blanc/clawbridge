# Agent Definition

`ClawAgent` is the main OpenClaw-style agent spec in `clawbridge`.

It groups three kinds of configuration:

## Shared OpenClaw fields

These fields describe the agent regardless of which builder you use:

- identity: `name`, `description`, `personality`, `role`
- model config: `provider`, `model`, API key wiring
  `provider` accepts first-class values like `anthropic`, `openai`, and `mistral`, plus arbitrary LiteLLM-compatible provider strings
- tools and skills: direct `ToolDefinition` values plus `ClawSkill`
- prompt composition: `system_prompt`, `additional_instructions`
- memory helpers: `memory_config`
- workspace loading: `workspace_path`, `workspace`
- sandbox and channel policy: `sandbox`, `channel_policy`

## Framework-specific extensions

These fields are meaningful, but their behavior depends on the selected framework:

- `storage`
- `knowledge`
- `channels`

Use them when you need native framework capabilities, but do not assume identical behavior across Agno and Agentica.

## Prompt assembly

The prompt builder composes identity, tooling, workspace context, memory, sandbox state, runtime state, and extra instructions in a fixed OpenClaw-style order.
