# Agno Backend

Agno is the most complete framework integration in the repository today.

## Build path

Use `build_agno_agent(...)` when you want a native Agno `Agent` from a `ClawAgent` or workspace-backed config.

```python
from clawbridge import build_agno_agent, load_agent_config

agent = load_agent_config("./agent.yaml")
native_agent = build_agno_agent(agent)
```

This path currently supports:

- OpenClaw prompt assembly from workspace files
- direct callable tools
- strict failure for schema-only skills without real callables
- native storage mapping for `storage`
- knowledge integration
- channel policy and Agno interface wiring

## Deploy path

Use Agno when you want the strongest deployment story in the repo today.

For native serving:

```bash
clawbridge serve --port 8000
```

For the optional filesystem-driven Agno deployment helper:

```bash
clawbridge init my-agno-helper
cd my-agno-helper
clawbridge dev
```

Both paths ultimately deploy through Agno `AgentOS`.

## What is Agno-specific

These features should be treated as Agno extensions rather than generic guarantees:

- database providers from `storage`
- vector knowledge ingestion from `knowledge`
- Slack and WhatsApp interfaces from `channels`
- optional filesystem deployment helper via `clawbridge.app.ClawApp`

## Hermes-Like Features

The Agno backend maps `clawbridge` configuration to Agno 2.5's native capabilities:

| ClawAgent field | Agno parameter | Effect |
|---|---|---|
| `agent_memory_mode: automatic` | `update_memory_on_run=True` | Auto-extract memories |
| `agent_memory_mode: agentic` | `enable_agentic_memory=True` | Agent-controlled memory |
| `learning.enabled: true` | `learning=True` | Self-improvement loop |
| `session.search_past_sessions: true` | `search_past_sessions=True` | Cross-session recall |
| `session.enable_session_summaries: true` | `enable_session_summaries=True` | Automatic session summaries |
| `session.compress_tool_results: true` | `compress_tool_results=True` | Context compression |
| `session.reasoning: true` | `reasoning=True` | Reasoning mode |

All features require `storage.enabled: true` with a persistent database (sqlite or postgres) for the Agno backend to store state.
