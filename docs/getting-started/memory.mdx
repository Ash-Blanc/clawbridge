# Add Memory

clawbridge has two memory layers: `ClawMemory` for lightweight in-process key-value storage, and native persistent memory backed by SQLite or PostgreSQL for real cross-session recall.

## ClawMemory

`ClawMemory` is a simple in-process store for facts and preferences. Good for small amounts of context you want to thread through a session without a database.

```python
from clawbridge import ClawMemory

memory = ClawMemory()
memory.remember("user_name", "Alice", category="preference")
memory.remember("timezone", "UTC", category="preference")

print(memory.recall("user_name"))
# → "Alice"
```

Use `ClawMemory` for:

- user preferences within a session
- known facts you want to inject into prompts
- lightweight context threading

## Going further: Persistent memory

`ClawMemory` doesn't survive process restarts. For real cross-session memory that works like [Hermes-agent](https://github.com/nousresearch/hermes-agent), use `agent_memory_mode` with a `StorageConfig`:

```python
from clawbridge import ClawAgent, AgentMemoryMode, StorageConfig, build_agno_agent

agent = ClawAgent(
    name="Molty",
    storage=StorageConfig(enabled=True, type="sqlite", db_url="agent.db"),
    agent_memory_mode=AgentMemoryMode.AUTOMATIC,
)

native_agent = build_agno_agent(agent)
# The agent now automatically extracts and recalls facts across sessions
native_agent.print_response("My favorite language is Python.")

# Next session — the agent remembers:
native_agent.print_response("What's my favorite language?")
# → "Your favorite language is Python."
```

### Memory modes

| Mode | Behavior |
|---|---|
| `AUTOMATIC` | Agent decides when to store and retrieve memories |
| `ALWAYS` | Store and retrieve on every turn |
| `NEVER` | Disable persistent memory |

### YAML equivalent

```yaml
storage:
  enabled: true
  type: sqlite
  db_url: agent.db
agent_memory_mode: automatic
```
