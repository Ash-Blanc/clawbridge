# Memory

There are three memory layers in `clawbridge`:

## ClawMemory

`ClawMemory` is the lightweight memory helper used by `clawbridge` prompt assembly and runtime wrappers.

It provides:

- long-term key/value memory
- short-term conversation history
- prompt-time memory summaries

```python
from clawbridge import ClawMemory

memory = ClawMemory()
memory.remember("user_name", "Alice", category="preference")
print(memory.recall("user_name"))
```

Use `ClawMemory` for prompt-time and lightweight state.

## Framework-native storage

Some frameworks can map `ClawAgent.storage` into native storage primitives. Today Agno has native storage support; Agentica does not.

```yaml
storage:
  enabled: true
  type: sqlite
  db_url: agent.db
```

## Native persistent memory (Hermes-like)

For agents that remember user preferences, learn from interactions, and recall past sessions, use the Hermes-like configuration:

```yaml
storage:
  enabled: true
  type: sqlite
  db_url: agent.db
agent_memory_mode: automatic    # off | automatic | agentic
learning:
  enabled: true
session:
  search_past_sessions: true
  enable_session_summaries: true
```

`AgentMemoryMode.AUTOMATIC` auto-extracts facts from conversations. `AgentMemoryMode.AGENTIC` gives the agent full control over what it remembers.

All of these require `storage.enabled: true` with a persistent database (sqlite or postgres). See [Hermes Features](/docs/guides/hermes-features) for the full configuration guide.
