# Agentica Backend

Agentica is supported through `build_agentica_agent(...)`.

## What works

- `ClawAgent` prompt and workspace loading
- prompt assembly through `ClawAgent.build_system_prompt()`
- direct callable tools exposed into scope
- placeholder scope objects for schema-only skill tools
- shared `ClawMemory` object exposed as a scope bridge

## Important boundaries

Agentica support is real, but narrower than Agno support:

- `build_agentica_agent(...)` returns config rather than a long-lived native runtime object
- `storage`, `knowledge`, and `channels` are not mapped to native Agentica features
- execution semantics depend on Agentica's spawn and callable interaction model

Use Agentica when you want OpenClaw-style prompts and skills with live-object execution, not when you need parity with Agno-specific features.
