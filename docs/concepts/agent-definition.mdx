# Agent Definition

`ClawAgent` is the central configuration model. It defines everything about your agent in a framework-agnostic way, then builders compile it into native objects.

## Minimal definition

```python
from clawbridge import ClawAgent, build_agno_agent

agent = ClawAgent(name="Molty")
native_agent = build_agno_agent(agent)
```

Or in YAML:

```yaml
name: Molty
```

Every field has a sensible default. Start minimal, add fields as needed.

## Identity fields

```yaml
name: Molty
description: A personal research assistant
personality: Pragmatic and concise. Cites sources.
role: Research assistant
```

These fields are injected into the system prompt's Identity section.

## Model configuration

```yaml
model:
  provider: anthropic          # openai, anthropic, groq, mistral, deepseek, or any LiteLLM string
  model: claude-sonnet-4-20250514
  temperature: 0.7
  max_tokens: 4096
```

Agno reads API keys from standard environment variables (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, etc.) automatically.

## Skills and tools

Skills are loaded from directories. Tools are defined inline:

```yaml
skill_paths:
  - ./extra-skills
tools:
  - name: get_weather
    description: Get current weather
```

Tools defined in YAML are schema-only — you must provide the callable in Python via `ClawAgent.tools`. For most cases, use skill directories with `tools.py` instead.

## Workspace

```yaml
workspace_path: .
```

Points to an OpenClaw workspace directory. The builder loads `AGENTS.md`, `MEMORY.md`, skills, and other workspace files automatically.

## Hermes-like features

```yaml
agent_memory_mode: automatic   # off | automatic | agentic
learning:
  enabled: true
session:
  search_past_sessions: true
  enable_session_summaries: true
  compress_tool_results: true
  reasoning: true
  num_history_runs: 5
```

These require `storage.enabled: true` to function. See [Hermes Features](/docs/guides/hermes-features).

## Storage and knowledge

```yaml
storage:
  enabled: true
  type: sqlite                 # in_memory | sqlite | postgres
  db_url: agent.db
knowledge:
  enabled: true
  type: lancedb                # lancedb | pgvector
  db_url: ./knowledge_db
  sources:
    - https://example.com/docs
```

## Framework-specific fields

These work with Agno but not necessarily with other backends:

- `storage` — database for session history, memory, learning
- `knowledge` — vector knowledge base
- `channels` — messaging platform connections (Slack, WhatsApp)

## Full YAML example

```yaml
name: Molty
description: A self-improving personal assistant
personality: Helpful and learns from every interaction.
role: Personal assistant
model:
  provider: anthropic
  model: claude-sonnet-4-20250514
workspace_path: .
storage:
  enabled: true
  type: sqlite
  db_url: agent.db
agent_memory_mode: automatic
learning:
  enabled: true
session:
  search_past_sessions: true
  enable_session_summaries: true
```
