# Memory

There are two memory layers in `clawbridge`:

1. `ClawMemory`, which is portable and shared by `ClawBridge`
2. backend-native storage, which depends on the selected runtime

Native storage is configured in the agent definition:

```yaml
storage:
  enabled: true
  type: sqlite # in_memory | sqlite | postgres
  db_url: memory.db
```

Agno currently maps this to native `Db` providers. Other backends may ignore or partially implement it.
