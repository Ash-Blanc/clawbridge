# clawbridge

`clawbridge` is a Python meta-framework for **portable agent definitions**.

The core idea is simple:

1. Define an agent once with `ClawAgent`
2. Attach skills, tools, and memory
3. Run it through `ClawBridge` on a supported backend

The portable core is built from:

- `ClawAgent`
- `ClawBridge`
- `ClawSkill`
- `ClawMemory`
- `ToolDefinition`

## What clawbridge is not

`clawbridge` is not only an Agno app shell, and it does not claim feature-perfect parity across runtimes.

There are two layers in the project:

- **Portable core**: agent definition, skills, prompt assembly, memory object, and runtime wrapper
- **Backend-specific helpers**: direct compile helpers and optional app-mode DX

## Current emphasis

The primary workflow is the portable core. Agno app mode still exists, but it is a convenience layer for Agno deployments rather than the product's main identity.
