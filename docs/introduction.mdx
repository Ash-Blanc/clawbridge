# clawbridge

`clawbridge` is a Python toolkit for building [OpenClaw](https://github.com/openclawai/openclaw)-style agents inside frameworks you already use — primarily [Agno](https://github.com/agno-agi/agno) and Agentica. It handles workspace loading, skill resolution, prompt composition, and memory configuration so you can stay focused on what your agent does, not how it runs.

## Why clawbridge?

Most agent frameworks make you choose between portable agent definitions and framework-native execution. clawbridge gives you both:

- **OpenClaw workspace conventions** — define agents in YAML, organize skills in directories, keep workspace context in markdown files
- **[Hermes-agent](https://github.com/nousresearch/hermes-agent)-like features** — persistent memory, cross-session recall, and self-improving learning loops
- **Framework-native execution** — `build_agno_agent()` and `build_agno_team()` return real Agno objects you can run, serve, and extend

## What you can build

| What | How |
|---|---|
| A single agent with skills and memory | `build_agno_agent(ClawAgent(...))` |
| A self-improving agent with learning | `AgentMemoryMode.AUTOMATIC` + `LearningConfig` |
| A multi-agent team | `build_agno_team(TeamConfig(...))` |
| An Agentica-backed agent | `build_agentica_agent(ClawAgent(...))` |

## Main builders

```python
from clawbridge import build_agno_agent, build_agno_team, build_agentica_agent, load_agent_config
```

- `build_agno_agent(agent)` — compile a `ClawAgent` into a native Agno `Agent`
- `build_agno_team(team)` — compile a `TeamConfig` into a native Agno `Team`
- `build_agentica_agent(agent)` — compile a `ClawAgent` into an Agentica agent
- `load_agent_config(path)` — load a `ClawAgent` or `TeamConfig` from a YAML file

## Key features

**Persistent memory** — agents remember facts across sessions using SQLite or PostgreSQL. No manual session management.

**Learning** — with `LearningConfig`, agents extract summaries from past interactions and use them to improve future responses.

**Cross-session recall** — `SessionConfig(search_past_sessions=True)` lets the agent retrieve context from previous conversations automatically.

**Multi-agent teams** — `TeamConfig` supports four coordination modes: `coordinate`, `route`, `broadcast`, and `tasks`. Each member is a full `ClawAgent` with its own model, skills, and memory.

**OpenClaw workspaces** — `clawbridge scaffold ./my-workspace` creates the directory structure, starter YAML, and skills. The workspace files (`AGENTS.md`, `MEMORY.md`, `BOOTSTRAP.md`) become part of the agent's system prompt automatically.

**Skill precedence** — skills are resolved from the workspace with configurable gating and override rules. You can load skills globally or per-agent.
